# CatForge 竞品展示页界面设计规范（领导汇报 MVP）

> 版本：v1.0  
> 适用范围：CatForge 一个月内可展示 MVP  
> 页面对象：输入一个目标 SKU，输出 2-3 个核心竞品、选择理由、证据卡和 SOP 推导轨迹  
> 设计口径：**业务领导汇报页，不是算法调试台，不是甲方终端系统，不是 AI 聊天产品**

---

## 1. 页面定位

本页面的目标不是展示“系统跑了多少步骤”，而是展示：

> **给定一个 SKU，系统如何从不同价值战场中选择最值得业务关注的 2-3 个核心竞品，并用市场证据说明为什么是它们。**

以海信 85E7Q 为例，页面应回答四个问题：

1. 85E7Q 当前主要在哪些价值战场竞争？
2. 系统最终建议重点关注哪 2-3 个竞品？
3. 每个竞品分别代表什么竞争压力？
4. 这些选择是否有价格、渠道、参数、卖点、销量、评论等证据支撑？

页面背后可以遵循 SOP 中的 M08-M15 链路：

```text
SKU 综合信号画像
→ 用户任务
→ 目标客群
→ 价值战场
→ 候选池召回
→ 竞品组件评分
→ 三槽位核心竞品选择
→ 证据卡与高层报告
```

但页面不能把 SOP 机械摊开。领导看到的应是**结论先行、证据支撑、可展开推导**。

---

## 2. 设计风格要求

### 2.1 总体风格

建议采用：

> **Google Workspace / Material Design 倾向的商用数据产品风格**

关键词：

```text
克制
清爽
可信
专业
留白充分
层级明确
数据友好
报告感强
```

不要做成：

```text
AI 炫技风
科幻风
大屏指挥舱风
黑底荧光风
机器人/魔法/大脑图标风
过度渐变和发光效果
```

### 2.2 视觉目标

页面给业务领导的感受应该是：

| 感受 | 说明 |
|---|---|
| 可信 | 每个结论都有证据入口 |
| 清楚 | 一眼知道核心竞品是谁 |
| 专业 | 用市场分析语言，不用模型术语堆砌 |
| 克制 | 不靠炫酷动画和 AI 图形营造智能感 |
| 可追溯 | 能展开看到 SOP 推导和证据链 |
| 可决策 | 每个竞品都有策略含义 |

## 3. 颜色规范

### 3.1 基础色

建议使用接近 Google 商用产品的中性色和蓝色体系。

```css
--color-bg-page: #F8FAFD;
--color-bg-card: #FFFFFF;
--color-bg-subtle: #F1F3F4;

--color-text-primary: #202124;
--color-text-secondary: #5F6368;
--color-text-muted: #80868B;

--color-border: #DADCE0;
--color-border-light: #E8EAED;

--color-primary: #1A73E8;
--color-primary-hover: #185ABC;
--color-primary-light: #E8F0FE;
```

### 3.2 竞品角色色

三类竞品用轻量标签色，不要大面积铺色。

| 竞品角色 | 主色 | 背景色 | 用法 |
|---|---:|---:|---|
| 正面对打竞品 | `#1A73E8` | `#E8F0FE` | 同层直接竞争 |
| 价格/销量挤压竞品 | `#F29900` | `#FEF7E0` | 价格压力、销量压力 |
| 高端标杆/潜在下探竞品 | `#9334E6` | `#F3E8FD` | 上方标杆、下探风险 |
| 需复核/证据不足 | `#D93025` | `#FCE8E6` | 风险提示 |
| 已验证/高置信 | `#188038` | `#E6F4EA` | 高可信 |

### 3.3 禁止色彩

避免：

```text
大面积黑色背景
霓虹蓝/紫
高饱和渐变
红绿强对比大面积堆叠
类似游戏 HUD 的发光线条
```

## 4. 字体与排版

### 4.1 字体

建议使用系统字体：

```css
font-family:
  Inter,
  Roboto,
  "Noto Sans SC",
  "PingFang SC",
  "Microsoft YaHei",
  Arial,
  sans-serif;
```

如果环境不允许安装字体，直接使用系统默认即可。

### 4.2 字号层级

| 用途 | 字号 | 字重 |
|---|---:|---:|
| 页面标题 | 24-28px | 600 |
| 区块标题 | 18-20px | 600 |
| 卡片标题 | 16px | 600 |
| 正文 | 14px | 400 |
| 次要说明 | 13px | 400 |
| 标签/注释 | 12px | 500 |
| 数字 KPI | 24-32px | 600 |

### 4.3 文案排版

业务判断类文案不要超过两行。  
长理由拆成：

```text
一句话理由
关键证据
关键差异
策略含义
```

不要把算法公式直接放在主页面。公式可以进入“查看推导”抽屉。

```text
价值战场摘要：左侧战场卡，右侧证据矩阵

```

### 5.3 圆角和阴影

```css
  0 1px 3px rgba(60, 64, 67, 0.08);
```

## 7. 页面 A：顶部输入与目标 SKU 结论

### 7.1 输入区

布局：

```text
[输入型号 / SKU 编码] [分析按钮] [数据版本] [最近更新时间]
```

示例：

```text
输入：85E7Q
按钮：生成核心竞品
数据版本：TV_2026W21
更新时间：2026-05-25
```

### 7.2 目标 SKU 结论条

使用浅蓝背景的提示条，不要用 AI 对话气泡。

示例：

```text
海信 85E7Q 的核心竞争不在全市场 85 吋电视，而在“高端画质”和“家庭观影改善”两个主价值战场。系统建议重点关注 3 个竞品，分别代表正面对打、价格/销量挤压和高端标杆/潜在下探压力。
```

样式：

```css
background: #E8F0FE;
border-left: 4px solid #1A73E8;
border-radius: 12px;
padding: 16px 20px;
```

## 8. 页面 B：目标 SKU 市场信号卡

### 8.1 目标

让领导先理解 85E7Q 是一个什么 SKU。

### 8.2 卡片内容

建议 6 个信号卡：

| 卡片 | 内容 |
|---|---|
| 产品形态 | 85 吋 / Mini LED / UHD |
| 价格位置 | 约 9000-10000 元段 |
| 主销渠道 | 线上 / 专业电商 |
| 核心卖点 | Mini LED、高亮度、分区控光、高刷、HDMI2.1 |
| 用户感知 | 画质、看球、接口、系统流畅、老人易用 |
| 主价值战场 | 高端画质、家庭观影改善 |

### 8.3 示例展示

```text
产品形态
85 吋 Mini LED 高端画质大屏

核心卖点
Mini LED / 5200nits / 3500分区 / 高刷 / HDMI2.1

用户感知
画质正向、看球体验、接口够用、老人能上手
```

## 9. 页面 C：核心竞品卡

### 9.1 设计原则

核心竞品区是页面主视觉。  
不要做普通表格，使用三张业务角色卡。

```text
┌──────────────┬──────────────┬──────────────┐
│ 正面对打竞品 │ 价格/销量挤压 │ 高端标杆/下探 │
└──────────────┴──────────────┴──────────────┘
```

如果系统只选出 2 个，则第三张显示：

```text
暂无高置信候选
原因：高端下探候选样本不足 / 需专家复核
```

不要硬凑。

---

### 9.2 竞品卡字段

每张卡必须包括：

| 字段 | 说明 |
|---|---|
| 竞品角色 | 正面对打 / 价格挤压 / 高端标杆 |
| 竞品型号 | 品牌 + 型号 |
| 主要战场 | 高端画质 / 大屏换新等 |
| 竞争压力 | 高 / 中高 / 中 / 需复核 |
| 一句话理由 | 面向业务领导 |
| 五维证据 | 价格、渠道、参数、卖点、市场/评论 |
| 关键差异 | 目标 SKU 优势和竞品优势 |
| 策略含义 | 对定价/卖点/防守/上探的启示 |
| 置信度 | 高/中/低 |
| 查看证据 | 进入证据抽屉 |
| 查看推导 | 进入 SOP 轨迹 |

---

### 9.3 正面对打竞品卡模板

```text
正面对打竞品
{品牌} {型号}

高端画质战场 / 家庭观影改善战场
竞争压力：高

一句话理由：
它与 85E7Q 在尺寸、价格、渠道和高端画质卖点上高度接近，是用户在同一预算下最可能同时比较的型号。

关键证据：
价格接近：均价差距 3.2%
渠道重合：专业电商重合度 0.78
参数接近：85吋 / Mini LED / 高刷
卖点接近：高亮度 / 分区控光 / HDMI2.1
市场有效：同池销量分位前 30%

关键差异：
85E7Q 优势：分区控光和画质表达更强
竞品优势：价格略低，销量更强

策略含义：
定价上应重点监控该竞品价格变化；卖点表达上突出 85E7Q 的画质证据。
```

---

### 9.4 价格/销量挤压竞品卡模板

```text
价格/销量挤压竞品
{品牌} {型号}

大屏换新战场 / 家庭观影改善战场
竞争压力：中高

一句话理由：
它不一定与 85E7Q 配置完全相同，但能够满足大屏换新和家庭观影任务，并通过更低价格或更强销量对 85E7Q 形成替代压力。

关键证据：
价格优势：低于 85E7Q 18%
销量强度：同尺寸同渠道销量分位前 15%
任务相似：大屏换新 / 家庭观影
渠道重合：线上渠道主销
评论支撑：尺寸满意、性价比相关评论较强

关键差异：
85E7Q 优势：高端画质卖点更完整
竞品优势：价格更低，换新吸引力更强

策略含义：
该竞品决定 85E7Q 的价格防守边界。若其促销加强，85E7Q 需要用画质证据或权益包守住价差。
```

---

### 9.5 高端标杆/潜在下探竞品卡模板

```text
高端标杆 / 潜在下探竞品
{品牌} {型号}

高端画质战场
竞争压力：中高

一句话理由：
它在高端画质参数或品牌势能上强于 85E7Q，价格通常更高；若近期促销下探进入相近价带，将对 85E7Q 形成上方压制。

关键证据：
参数优势：亮度 / 分区 / OLED / 色彩优于目标 SKU
高端战场：同属高端画质战场
价格锚点：高于 85E7Q 12%
下探风险：近期降价 9%
销额有效：同池销额分位前 20%

关键差异：
85E7Q 优势：价格更低，性价比更强
竞品优势：高端参数或品牌认知更强

策略含义：
该竞品用于判断 85E7Q 是否有上探空间，以及大促期间是否需要提前防御高端机型下探。
```

## 10. 页面 D：价值战场摘要

### 10.1 目标

让领导理解为什么不是全市场找竞品，而是按价值战场找。

### 10.2 展示方式

使用横向战场卡片或轻量矩阵。

| 价值战场 | 关系 | 85E7Q 依据 | 竞品选择作用 |
|---|---|---|---|
| 高端画质战场 | 主战场 | Mini LED、高亮度、分区控光 | 找正面对打和高端标杆 |
| 家庭观影改善战场 | 主/次战场 | 85 吋、音画、评论画质 | 找观影改善型竞品 |
| 游戏电视战场 | 次战场 | 高刷、HDMI2.1、看球评论 | 找游戏/运动型竞品 |
| 大屏换新战场 | 机会战场 | 85 吋，但价格非低价 | 找价格挤压型竞品 |
| 家庭护眼战场 | 弱战场 | 有护眼宣传，评论不足 | 不作为核心竞品筛选主线 |

### 10.3 视觉建议

每个战场用一张小卡：

```text
高端画质战场
主战场
得分：84
依据：Mini LED / 高亮 / 分区 / 画质评论
```

不要做复杂网络图。复杂图谱不适合一月 MVP 和领导汇报。

## 11. 页面 E：证据矩阵

### 11.1 目标

让每个竞品选择理由显得专业、可信、可追溯。

### 11.2 证据矩阵结构

建议每个竞品展开后显示：

| 证据类型 | 指标 | 解释 |
|---|---|---|
| 价格证据 | 价格差距、价格带、降价趋势 | 是否构成价格压力 |
| 渠道证据 | 渠道重合度、主销渠道 | 是否在同一场域竞争 |
| 参数证据 | 尺寸、Mini LED、亮度、分区、高刷、HDMI | 产品力是否可比 |
| 卖点证据 | 标准卖点相似度、核心卖点重合 | 用户看到的价值是否相近 |
| 市场证据 | 销量分位、销额分位、增长趋势 | 是否有真实市场压力 |
| 评论证据 | 画质、游戏、易用、护眼等评论主题 | 用户是否感知相关卖点 |

### 11.3 证据强度显示

使用轻量进度条，不用仪表盘。

```text
价格证据       强    92
渠道证据       中高  78
参数证据       强    81
卖点证据       强    84
市场证据       中高  70
评论证据       中    65
```

## 12. 页面 F：SOP 推导轨迹

### 12.1 展示原则

SOP 推导作为抽屉或展开区域，不放在主屏。

标题：

```text
为什么系统选择这个竞品？
```

展示 7 步即可：

```text
① SKU 信号画像
② 用户任务识别
③ 目标客群判断
④ 价值战场判定
⑤ 候选池召回
⑥ 组件评分
⑦ 三槽位选择
```

### 12.2 示例

```text
① SKU 信号画像
85E7Q：85吋 / Mini LED / 高亮度 / 高刷 / 专业电商 / 画质评论

② 用户任务识别
高画质影音、家庭观影改善、体育观看

③ 目标客群判断
高端改善用户、家庭观影改善用户、游戏/体育用户

④ 价值战场判定
主战场：高端画质、家庭观影
次战场：游戏电视
机会战场：大屏换新

⑤ 候选池召回
从全量 SKU 中召回同战场、同价格、同渠道、同任务候选 42 个

⑥ 组件评分
价格、渠道、尺寸、参数、卖点、任务、销量、趋势综合评分

⑦ 三槽位选择
从候选中选择正面对打、价格挤压、高端标杆三个角色
```

### 12.3 不要展示

不要展示：

```text
完整 M00-M16
完整 SQL
完整公式堆叠
大段 JSON
LLM 思考过程
```

公式可在“技术详情”二级抽屉中展示。

## 13. 页面 G：候选池与未选原因

### 13.1 目标

体现专业性：系统不仅知道为什么选，也知道为什么不选。

### 13.2 展示方式

放在页面底部折叠区：

```text
查看候选池与未选原因
```

表格字段：

| 候选型号 | 候选角色 | 综合分 | 未选原因 |
|---|---|---:|---|
| 候选 D | 正面对打 | 0.81 | 与已选正面对打竞品高度重复，业务信息增量不足 |
| 候选 E | 价格挤压 | 0.76 | 价格有优势，但渠道重合度低 |
| 候选 F | 标杆 | 0.74 | 参数更强，但销量过低，市场压力不足 |
| 候选 G | 潜在 | 0.71 | 降价趋势不明显，暂不作为核心竞品 |

这会明显提升领导对系统的信任度。

## 14. 报告导出页

### 14.1 目标

支持领导汇报使用。

按钮：

```text
导出竞品分析报告
导出竞品证据卡
复制汇报摘要
```

### 14.2 导出内容

报告建议包含：

```text
目标 SKU 概览
价值战场判断
核心 2-3 竞品
每个竞品选择理由
证据矩阵
策略启示
候选池摘要
数据版本和时间
```

### 14.3 汇报摘要模板

```text
海信 85E7Q 的主要竞争压力来自三个方向：
第一，{竞品A} 是同价位高端画质战场中的正面对打竞品；
第二，{竞品B} 以更低价格和更强销量覆盖大屏换新任务，形成价格/销量挤压；
第三，{竞品C} 在高端画质参数上更强，若价格下探将压缩 85E7Q 的上探空间。
以上判断基于价格、渠道、参数、卖点、销量和评论证据。
```

## 15. 交互规范

### 15.1 页面默认展开

默认只展示：

```text
目标 SKU 结论
目标 SKU 信号卡
核心 2-3 竞品卡
```

### 15.2 二级展开

点击竞品卡：

```text
打开右侧抽屉
显示证据矩阵、关键差异、SOP 推导
```

### 15.3 三级展开

点击证据项：

```text
显示原始 evidence
例如：原始参数、宣传句、评论片段、量价指标
```

### 15.4 动效

动效要克制：

```text
卡片 hover 轻微阴影
抽屉滑入
折叠展开
loading skeleton
```

不要使用：

```text
粒子动画
AI 打字机效果
机器人正在思考
光效扫描
动态脑图
```

## 16. 空状态和低置信状态

### 16.1 数据不足

如果无法选满 3 个竞品：

```text
高端标杆/潜在下探竞品暂无高置信候选

原因：
- 同战场样本不足
- 高端型号近期无有效量价数据
- 候选证据不完整

建议：
- 扩大价格范围
- 纳入最近 12 周促销数据
- 进入专家复核
```

### 16.2 低置信

显示黄色或灰色提示：

```text
该竞品选择置信度为中，需要业务复核。
主要原因：评论证据不足，且价格走势样本较少。
```

不要用红色大面积警告，除非数据严重缺失。

## 19. 文案规范

### 19.1 应使用的表达

```text
正面对打竞品
价格/销量挤压竞品
高端标杆/潜在下探竞品
竞争压力
价值战场
选择理由
证据摘要
关键差异
策略启示
需复核
样本不足
```

### 19.2 避免使用的表达

```text
AI 认为
模型猜测
神经网络分析
智能涌现
向量空间结果
黑盒判断
魔法生成
```

如果必须展示模型或 LLM，放到技术详情，不放在领导页主文案。

### 19.3 解释口径

不要说：

```text
claim_similarity = 0.83，因此选它。
```

要说：

```text
该型号与 85E7Q 同样强调 Mini LED、高亮度和分区控光，且价格和渠道接近，因此是高端画质战场中的正面对打竞品。
```

## 21. Codex 实施提示

给 Codex 的目标可以这样写：

```markdown
Goal:
Redesign the competitor display page as a business-leadership report page.

Product style:
Use a clean Google Workspace / Material-like business UI. Avoid heavy AI/robot/sci-fi styling. This is a credible business decision page, not an AI demo page.

Page purpose:
Given a SKU, show 2-3 core competitors selected from value battlefields. Each competitor must show its business role, one-sentence reason, evidence matrix, key differences, strategy implication, and expandable SOP trace.

Required sections:
1. SKU search and executive conclusion banner.
2. Target SKU signal strip.
3. Readiness KPI row.
4. Three role-based competitor cards:
   - direct competitor
   - price/sales pressure competitor
   - benchmark/potential downshift competitor
5. Battlefield summary.
6. Evidence drawer.
7. SOP trace timeline.
8. Candidate pool accordion with not-selected reasons.
9. Report export panel.

Do not:
- Build AI chat UI.
- Use dark neon dashboard styling.
- Show only raw scores.
- Show dozens of competitors.
- Expose M00-M16 as the main visual flow.
- Use “AI thinks” language.

Acceptance:
- A business leader can understand the top 2-3 competitors within 30 seconds.
- Each competitor has a clear role and business reason.
- Each reason is backed by price/channel/param/claim/market/comment evidence.
- SOP trace is available but secondary.
- Page looks like a clean enterprise product, not an AI gimmick.
```

## 22. 验收标准

### 22.1 30 秒理解测试

找一个没有参与项目的业务同事，看页面 30 秒后应能回答：

```text
85E7Q 的 2-3 个核心竞品是谁？
每个竞品代表什么压力？
为什么选它？
```

### 22.2 证据完整性测试

每个竞品卡至少要有：

```text
价格证据
渠道证据
参数证据
卖点证据
市场证据
评论证据或评论不足说明
```

### 22.3 专业感测试

页面不能出现：

```text
大段 JSON
算法变量名
LLM 思考过程
无解释分数
十几个竞品列表
AI 炫技元素
```

### 22.4 业务行动测试

每个竞品必须有策略启示：

```text
定价防守
卖点强化
价格下限压力
上方标杆压力
促销风险
```

## 23. 最终效果目标

页面最终应该让领导觉得：

> 这不是一个简单的竞品相似度页面，而是一个能够从市场证据中提炼竞争压力的专业分析工具。

最重要的一句话：

> **系统不是给 85E7Q 找一串相似型号，而是识别三类真实竞争压力：谁正面对打、谁用低价或销量挤压、谁从高端下探压制。每个判断都有价格、渠道、参数、卖点、销量和评论证据。**
