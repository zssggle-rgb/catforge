# 05 SKU 画像反推、任务客群战场与竞品推导

## 1. 反推思路

SKU 画像不是把预制标签贴到型号上，而是从真实数据反推：

```text
这个 SKU 有哪些可验证参数
  -> 这些参数支撑哪些标准卖点
  -> 宣传和评论是否验证这些卖点
  -> 这些卖点解决哪些用户任务
  -> 这些任务对应哪些目标客群
  -> 任务、客群、卖点和市场表现共同形成哪些价值战场
  -> 在同战场、同价格/尺寸/渠道中，哪些 SKU 与它竞争
```

## 2. SKU 市场画像

输入：`core3_clean_market_fact`

输出：`core3_sku_market_profile`

核心指标：

```text
sales_volume_total
sales_amount_total
price_wavg
price_latest
price_trend
sales_trend
channel_share
price_percentile_in_size_band
sales_percentile_in_size_band
amount_percentile_in_size_band
sample_sufficiency
```

同尺寸/相邻尺寸可比池：

```text
screen_size_inch within +/- 10 inch
或同官方尺寸段：65/75/85/98/100
```

价格带：

```text
低价位
主流价位
中高价位
高端价位
```

价格带不是固定绝对值，优先按当前数据分位和尺寸池计算。

## 3. SKU 参数画像

输入：`core3_extract_param_value`

输出：`core3_sku_semantic_profile.standard_params_json`

核心参数分组：

| 组 | 参数 |
| --- | --- |
| 基础显示 | 尺寸、分辨率、面板类型 |
| 画质显示 | 峰值亮度、背光技术、分区数、色域、HDR |
| 运动游戏 | 刷新率、HDMI2.1、VRR、ALLM、输入延迟 |
| 音响沉浸 | 扬声器功率、声道、杜比、低音 |
| 智能系统 | RAM、ROM、芯片、AI、语音 |
| 护眼易用 | 低蓝光、无频闪、护眼认证、长辈模式 |
| 外观安装 | 机身、边框、挂装、家居适配 |

输出必须保留：

- 标准值。
- 原始值。
- 来源优先级。
- 置信度。
- 冲突。
- evidence。

## 4. SKU 卖点画像

输入：

- 参数画像。
- 卖点句命中。
- 评论主题汇总。

输出：`core3_sku_claim_activation`

每个卖点：

```json
{
  "卖点": "高刷新率流畅体验",
  "激活分": 0.84,
  "参数证据": ["刷新率 144Hz"],
  "宣传证据": ["高刷相关宣传句"],
  "评论证据": ["看球不卡等评论"],
  "缺失信号": [],
  "置信度": 0.78
}
```

页面展示时不出现 `CLAIM_HIGH_REFRESH_RATE`，只显示“高刷新率流畅体验”。

## 5. SKU 评论画像

输入：`core3_extract_comment_topic_hit`

输出：`core3_sku_comment_topic_summary`

汇总：

- 用户最关注的正向体验。
- 用户最关注的负向风险。
- 产品体验与服务体验分离。
- 样例句。
- 评论数量和置信度。

示例业务表达：

```text
用户反馈主要集中在“画质清晰”“屏幕大”“安装服务”，其中画质相关正向反馈占比较高；游戏流畅相关反馈较少，因此游戏体育战场只能由参数和宣传支撑，评论验证不足。
```

## 6. 用户任务反推

任务得分公式：

```text
task_score =
  claim_signal * claim_weight
  + param_signal * param_weight
  + comment_signal * comment_weight
  + market_signal * market_weight
```

默认权重：

```text
卖点 0.40
参数 0.25
评论 0.20
市场 0.15
```

示例：游戏娱乐任务

```text
卖点信号：高刷新率、HDMI2.1、低延迟
参数信号：刷新率、接口、VRR、延迟
评论信号：游戏流畅、看球不卡
市场信号：线上渠道、同高刷池销量表现
```

输出：

- 任务分。
- 关系级别：主要/次要/弱相关。
- 支撑证据。
- 缺失信号。

## 7. 目标客群反推

客群由任务和市场位置推导：

```text
group_score =
  source_task_score * 0.60
  + price_channel_fit * 0.20
  + comment_validation * 0.20
```

示例：

```text
游戏用户 =
  游戏娱乐任务 * 0.70
  + 线上渠道适配 * 0.15
  + 中高价性能价格带 * 0.15
```

业务页面表达：

```text
系统判断该型号主要面向“大屏观影升级用户”，次要覆盖“游戏体育用户”。判断依据是 85 英寸大屏、高亮画质和家庭观影评论更强，游戏相关评论验证不足。
```

## 8. 价值战场反推

价值战场必须同时包含语义分和市场分：

```text
battlefield_score =
  semantic_score * semantic_weight
  + market_score * market_weight
```

语义分：

```text
核心任务 0.40
核心卖点组合 0.35
评论验证 0.15
目标客群 0.10
```

市场分：

```text
价格位置 0.30
销量验证 0.25
渠道适配 0.15
趋势信号 0.15
可比池密度 0.15
```

### 8.1 如何回答“为什么是游戏体育战场”

不能只说“因为有游戏体育战场”。要逐层回答：

1. 目标 SKU 是否具备高刷、接口、低延迟或运动画质相关信号。
2. 目标 SKU 是否有游戏、体育、运动流畅的评论或宣传验证。
3. 目标 SKU 在同尺寸/同价位中是否有销量或渠道支持。
4. 候选 SKU 是否也在同一战场得分高。
5. 两者是否在同一用户任务和客群上重合。

如果第 2 或第 3 不足，页面应说：

```text
游戏体育是次要机会战场，不是当前主战场。
```

## 9. 候选池召回

基础候选：

```text
同类目所有 SKU - 目标 SKU
```

不排除同品牌。

召回分层：

### 9.1 可比基础池

- 同类目。
- 有可用市场或参数数据。
- 尺寸可比。

### 9.2 任务/战场池

- 与目标 SKU 有共同主要或次要任务。
- 与目标 SKU 有共同价值战场。

### 9.3 压力池

- 价格更低但任务相似。
- 销量更强。
- 价格近期下探。
- 渠道重合。

### 9.4 标杆池

- 参数更强。
- 卖点更完整。
- 战场分更高。
- 高价或高销额代表性更强。

## 10. 组件分

组件分全部 0 到 1。

| 组件 | 说明 |
| --- | --- |
| 尺寸相似 | 同尺寸最高，相邻尺寸次之 |
| 价格相似 | 与目标加权均价接近 |
| 价格压力 | 候选低价或降价形成挤压 |
| 渠道重合 | 渠道销售占比重叠 |
| 销量强度 | 候选销量或销额相对强 |
| 参数相似 | 核心参数同档位 |
| 参数优势 | 候选在关键参数更强 |
| 卖点相似 | 标准卖点激活重合 |
| 评论主题相似 | 用户关注点重合 |
| 任务相似 | 用户任务得分重合 |
| 战场相似 | 价值战场得分重合 |

## 11. 三槽位选择

### 11.1 正面对打

回答：

```text
谁和目标 SKU 最像，并且抢同一批用户。
```

必要证据：

- 同尺寸或相邻尺寸。
- 价格接近。
- 主任务或主战场重合。
- 核心卖点相似。
- 渠道有重合。

### 11.2 价格/销量挤压

回答：

```text
谁通过价格、销量或渠道声量压制目标 SKU。
```

必要证据：

- 任务相似。
- 价格更有吸引力或销量更强。
- 渠道重合。
- 不是完全不可比产品。

### 11.3 高端标杆/潜在下探

回答：

```text
谁代表更高竞争标杆，或一旦降价就会进入目标价格带。
```

必要证据：

- 关键参数更强。
- 战场分更高。
- 销售额或品牌/产品位置更强。
- 价格差距未大到完全不可比，或已有下探趋势。

## 12. 竞品解释结构

页面先给结论，再解释推导：

```text
系统将 85E5Q-PRO 识别为 85E7Q 的正面对打竞品。

第一，它们在可比基础池中成立：同为 85 英寸，价格带接近，线上渠道重合。
第二，它们抢同一类任务：大屏家庭观影和高画质升级均为主要任务。
第三，它们处在同一主战场：高亮画质/大屏观影战场分均达到主要关系。
第四，85E5Q-PRO 的销量或销额表现更强，会对 85E7Q 形成直接分流。
```

如果证据不足：

```text
该槽位暂无高置信竞品。原因是当前样例数据中目标型号缺少结构化卖点，且同尺寸候选的评论主题覆盖不足。
```

## 13. 验收

1. 任何 SKU 的任务、客群、战场都不是按型号写死。
2. 修改某个 SKU 的评论主题，应影响该 SKU 的任务和战场分。
3. 修改价格或销量，应影响市场画像和竞品槽位分。
4. 同品牌 SKU 可以进入竞品结果。
5. 页面能解释“目标主战场是什么”和“候选主战场是否相同”。
6. 没有证据时输出不足原因，不硬凑。
