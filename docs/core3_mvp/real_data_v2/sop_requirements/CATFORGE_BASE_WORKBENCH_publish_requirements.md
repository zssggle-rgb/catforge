# CatForge 小奥家电市场分析工作台发布层需求

## 1. 背景

当前 CatForge 已经形成从数据清洗、事实画像、语义画像、市场图谱到小奥问答的完整链路：

```text
M00/M01/M02
  -> M03B/M04C/M05C/M07
  -> M09C/M10C/M11C/M11D/M12C
  -> catforge_analyst CLI
  -> 小奥飞书问答 / 飞书文档报告
```

这个链路适合做计算、追溯和专家问答，但业务用户还需要一个稳定的“分析结果工作台”：

- 不通过数据库或命令行，也能浏览 SKU、价值战场、竞品关系和用户卖点价值。
- 不每次都等智能体重新组织结果，常用结论可以直接在结构化视图中查看。
- 方便业务团队筛选、复核、分享和补充备注。
- 未来可以作为飞书卡片、飞书文档、看板和外部前端的共同结果层。

因此需要新增一个 **CatForge 发布层**，把已经生成的核心分析结果同步到飞书多维表格，形成 **小奥家电市场分析工作台**。

## 2. 模块定位

多维表格工作台是 CatForge 的业务消费层，不是计算源头，也不是原始数据仓库。

```text
PostgreSQL 分析结果库
  -> CatForge 发布服务
  -> 飞书多维表格工作台
  -> 小奥问答 / 业务浏览 / 飞书文档报告入口
```

边界：

- PostgreSQL 仍然是事实层和计算层的可信数据源。
- M00-M12C 仍然负责生成清洗、画像、图谱、竞品和卖点价值结果。
- 多维表格只承载对业务用户有用的摘要、索引、链接和复核状态。
- 不把原始评论、原始量价明细、大 JSON、LLM 原始响应或内部调试字段发布到工作台。

## 3. 业务目标

### 3.1 面向的问题

工作台必须支持业务用户不用写 SQL 也能回答：

- 当前 TV 品类有哪些可分析 SKU？
- 某个 SKU 的市场位置、主战场、主任务、主客群是什么？
- 每个价值战场覆盖哪些 SKU，市场空间有多大？
- 某个 SKU 的重点竞品是谁，为什么是这些竞品？
- 某个 SKU 的用户卖点价值有哪些，哪些是高溢价、门槛、待激活或竞品拦截？
- 哪些分析已经同步，哪些结果缺失或需要重新生成？
- 小奥回答里引用的飞书详细报告在哪里？

### 3.2 业务用户

| 用户 | 典型诉求 |
| --- | --- |
| 产品经理 | 查看 SKU 在尺寸、价格、价值战场、卖点和竞品上的结构位置。 |
| 市场负责人 | 快速看价值战场规模、重点竞品、卖点价值和机会缺口。 |
| 渠道/导购策略人员 | 查看同池竞品、用户支付理由和导购表达重点。 |
| 分析人员 | 核对同步状态、追溯报告链接、检查异常结果。 |
| 小奥智能体 | 通过稳定索引获取报告入口、候选结果和业务摘要。 |

## 4. 一期范围

一期只发布最核心、复用最高的五类结果表。

| 表 | 业务作用 |
| --- | --- |
| 分析批次表 | 说明本次工作台数据来自哪个分析批次、覆盖哪些 SKU、同步是否成功。 |
| SKU 总览表 | 每个 SKU 一行，展示市场位置、主战场、主任务、主客群、报告入口。 |
| 价值战场图谱表 | 每个价值战场一行，展示覆盖 SKU、分配销量、市场空间和代表 SKU。 |
| 竞品关系表 | 每个目标 SKU 与重点竞品一行，展示竞品角色、替代压力和报告入口。 |
| 用户卖点价值表 | 每个 SKU 的重点用户卖点一行，展示卖点角色、可解释金额/销量和证据摘要。 |

一期不做复杂仪表盘，不做跨品类经营看板，不做原始明细审计台。先把结果稳定发布出来，让业务用户能看、能筛、能点开报告。

## 5. 非目标

本次不做以下事项：

- 不修改 M00-M12C 的核心计算逻辑。
- 不把飞书多维表格作为主数据库。
- 不同步原始评论全文。
- 不同步原始周度量价明细。
- 不同步原始参数 118 字段和原始卖点全文大表。
- 不同步完整证据原子、LLM prompt、LLM response 或内部 JSON。
- 不在多维表格中暴露 Mxx 模块名作为业务主字段。
- 不在一期创建复杂 Base 仪表盘或自动化流程。
- 不通过多维表格反向改写 PostgreSQL 分析结果。

## 6. 核心业务口径

### 6.1 工作台是发布缓存，不是事实源

工作台中的字段必须能追溯到当前分析批次，但业务结论以 PostgreSQL 中的最新计算结果为准。多维表格只用于展示、协作和轻量复核。

### 6.2 业务字段优先

字段命名应使用业务语言：

- 使用“主价值战场”“重点竞品”“用户卖点价值”“可解释金额”。
- 避免把 `battlefield_code`、`claim_code`、`relation_status` 等内部字段作为默认可见字段。
- 审计字段可以保留，但默认视图中隐藏。

### 6.3 只发布稳定摘要

每条记录应控制为业务可读摘要：

- SKU 总览表：一行看清 SKU 的市场位置和主要画像。
- 竞品关系表：一行看清某个竞品为什么入选。
- 用户卖点价值表：一行看清卖点角色、价值、证据和建议。

详细过程继续放入飞书文档报告，通过链接下钻。

### 6.4 同步必须幂等

同一批次、同一品类、同一业务主键重复同步，不得产生重复记录。更新后的记录应覆盖旧值，并保留同步时间。

## 9. CLI 与 Skill 需求

### 9.1 CLI

建议新增独立发布 CLI：`catforge_publish`。

原因：

- `catforge_analyst` 负责分析和问答。
- `catforge_pipeline` 负责执行分析链路。
- `catforge_publish` 负责把已生成结果发布到外部协作工具。

一期命令：

```bash
python -m app.cli.catforge_publish base init --category tv --base-name "小奥家电市场分析工作台"
python -m app.cli.catforge_publish base sync --scope sku-overview --category tv --batch-id latest
python -m app.cli.catforge_publish base sync --scope battlefield-map --category tv --batch-id latest
python -m app.cli.catforge_publish base sync --scope competitor-relations --category tv --batch-id latest
python -m app.cli.catforge_publish base sync --scope claim-value --category tv --batch-id latest
python -m app.cli.catforge_publish base sync-all --category tv --batch-id latest
python -m app.cli.catforge_publish base open --category tv
```

CLI 必须支持：

- `--dry-run`：只输出将同步的行数和目标表，不写入飞书。
- `--limit`：便于测试少量记录。
- `--allow-schema-update`：允许补字段；默认不自动破坏已有结构。
- `--format json|text`：自动化使用 JSON，人工使用 text。

### 9.2 小奥 Skill

小奥 Skill 后续应支持这些自然语言：

- “把最新电视分析结果同步到工作台。”
- “打开小奥家电市场分析工作台。”
- “同步海信电视的竞品关系到多维表格。”
- “重新发布用户卖点价值结果。”
- “查看工作台同步状态。”

Skill 只负责调用发布 CLI，不直接拼多维表格字段，不直接读取数据库。

## 10. 权限与分享需求

工作台用于业务协作，权限必须清晰：

- 创建者可以是 bot 或指定业务账号，但必须在配置中明确。
- 默认业务用户应能通过链接打开工作台。
- 若当前飞书租户限制“互联网获得链接的人可阅读”，CLI 必须给出明确提示，不得假装已开放。
- 同步失败时不能吞掉权限错误。

## 11. 数据量与性能要求

TV 一期数据量预估：

| 表 | 预计行数 |
| --- | ---: |
| 分析批次表 | 小于 100 |
| SKU 总览表 | 200-500 |
| 价值战场图谱表 | 12-50 |
| 竞品关系表 | 600-2000 |
| 用户卖点价值表 | 1000-4000 |

要求：

- 单次写入按飞书 API 能力分批，单批不超过 200 行。
- 同步过程不得长时间占用 205 高 CPU。
- 支持按 scope 分开同步，避免一次失败影响全部表。
- 支持断点重试，至少做到重复执行不产生重复记录。

## 12. 验收标准

### 12.1 文档验收

- 需求文档和详细设计文档均落盘。
- 表结构、唯一键、同步流程、CLI 和 Skill 边界明确。
- 明确工作台不是事实源，不替代 PostgreSQL。

### 12.2 功能验收

后续实现完成后，应满足：

- 能创建或复用飞书多维表格。
- 能生成五张一期表。
- 能将最新 TV 分析结果同步到五张表。
- 重复同步不会生成重复记录。
- 每条业务记录能追溯批次、品类、SKU 或维度。
- 业务视图不暴露内部模块名、命令、JSON 或调试字段。
- 小奥可以用自然语言触发同步和打开工作台。

### 12.3 数据验收

以 TV 现有 200+ SKU 为样本：

- SKU 总览行数与当前可分析 SKU 数一致。
- 价值战场图谱行数与当前战场 taxonomy 和图谱输出一致。
- 竞品关系 Top 3 与 `catforge_analyst competitor-set` 输出一致。
- 用户卖点价值与 M12C 最新口径一致。
- 报告链接可打开，权限状态可验证。
