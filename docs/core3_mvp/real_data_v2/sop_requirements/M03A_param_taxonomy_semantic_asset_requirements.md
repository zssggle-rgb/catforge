# M03A 品类参数语义资产生成 SOP 需求

## 0. 定位

M03A 是从原始参数字段生成品类参数语义资产的模块。它不生成 SKU 参数画像，不判断卖点是否成立，不生成用户任务、目标客群、价值战场或竞品结论。

M03A 解决的问题是：

```text
某个品类现有原始参数字段到底能表达哪些产品能力。
```

M03A 的输出是可版本化、可复核、可发布的品类参数 taxonomy。后续 M03B 使用已发布 taxonomy 生成 SKU 参数事实画像。

## 1. 核心原则

1. 程序内不内置任何品类标准参数。
2. 电视、空调、洗衣机、冰箱等品类分别生成自己的 taxonomy。
3. 旧标准参数只能作为参考资产，不能作为框架约束原始字段。
4. M03A 先统计原始字段，再用 LLM 进行语义归纳，最后由规则和人工复核发布。
5. M03A 生成参数语义能力标签，不生成最终卖点、用户任务、目标客群、价值战场标签。
6. 缺失、空值、`-`、unknown 不得被解释为 false。
7. 所有 LLM 结论必须带来源字段、样例值、置信度和复核状态。

## 2. 输入

| 输入 | 来源 | 用途 |
| --- | --- | --- |
| 原始参数 evidence | M02 `param_raw` | 统计字段原貌、覆盖率、值样例 |
| 字段覆盖统计 | M03A 规则统计 | 供 LLM 语义归纳使用 |
| 字段共现关系 | M03A 规则统计 | 辅助判断字段是否表达同一事实 |

当前实现只直接读取 M02 `param_raw` evidence。M01 清洗参数表通过 M02 evidence 的 `clean_field`、`clean_value`、`value_presence` 间接进入 M03A；旧标准参数和原始卖点文本不作为本版 M03A 输入，避免用既有框架反向限制原始字段归纳。

## 3. 输出

M03A 发布一个品类参数 taxonomy，例如：

```text
TV_PARAM_TAXONOMY_v20260620
AC_PARAM_TAXONOMY_v20260620
```

taxonomy 包含：

| 资产 | 说明 |
| --- | --- |
| 原始字段清单 | 当前品类有哪些原始参数字段、覆盖率和值形态 |
| 字段聚类 | 哪些字段可能表达同一类产品事实 |
| 标准参数定义 | 该品类可用的标准参数事实 |
| 字段映射规则 | 原始字段如何进入标准参数 |
| 参数语义能力标签 | 参数表达的产品能力、收益提示、场景提示 |
| 风险与复核项 | 高频未解释字段、弱信号、冲突、口径不明 |

## 4. 不做事项

M03A 不做：

- 不生成 SKU 参数事实画像。
- 不判断某个卖点是否被激活。
- 不判断某个参数支撑哪个具体用户任务。
- 不生成目标客群。
- 不生成价值战场。
- 不生成竞品候选或评分。
- 不把评论体验当硬参数。
- 不把电视 taxonomy 用于空调等其他品类。

## 5. 参数语义能力标签

M03A 输出的标签是底层能力语言，不是业务结论。

| 标签字段 | 含义 | 示例 |
| --- | --- | --- |
| `capability_tags` | 参数表达的产品能力 | `motion_smoothness`、`color_expression` |
| `benefit_hints` | 可能给用户带来的价值 | 运动画面更流畅、色彩更真实 |
| `scenario_hints` | 可能相关的使用场景 | 游戏、体育观看、家庭观影 |
| `comparison_axis` | 如何比较强弱 | 越高越强、支持/不支持、需分档 |
| `evidence_role` | 证据角色 | 强参数证据、辅助证据、弱信号 |
| `risk_notes` | 使用风险 | 字段口径不明、0 值需解释 |

禁止在 M03A 中输出：

```text
claim_code
task_code
target_group_code
battlefield_code
```

这些代码由 M04、M09、M10、M11 在各自模块中定义和映射。

## 6. LLM 使用要求

M03A 允许使用 LLM，但 LLM 只承担语义归纳，不承担事实确认。

LLM 可以做：

1. 根据字段名和值样例推断字段语义。
2. 将多个原始字段合并为标准参数候选。
3. 生成能力标签、收益提示、场景提示、风险提示。
4. 标记需要人工复核的字段或参数候选。

LLM 不可以做：

1. 凭空新增没有字段支持的标准参数。
2. 直接生成卖点、任务、客群、战场结论。
3. 把弱信号当强事实。
4. 覆盖规则统计结果。
5. 把一个 SKU 的样例推广成全品类结论。

部署配置通过环境变量注入，不写入仓库：

| 环境变量 | 用途 |
| --- | --- |
| `CATFORGE_LLM_BASE_URL` | OpenAI-compatible endpoint，例如 DeepSeek OpenAI 入口 |
| `CATFORGE_LLM_ANTHROPIC_BASE_URL` | Anthropic-compatible endpoint 预留配置 |
| `CATFORGE_LLM_API_KEY` | LLM API key，只在部署环境设置 |
| `CATFORGE_LLM_MODEL` | 模型名，当前使用 `deepseek-v4-pro` |
| `CATFORGE_LLM_TIMEOUT_SECONDS` | 请求超时时间 |

## 7. 发布门槛

taxonomy 发布前必须满足：

1. 当前品类原始字段 100% 有处理状态。
2. 高频字段不能处于未解释状态，除非人工 waiver。
3. active 标准参数必须有原始字段证据或明确人工依据。
4. 没有字段支持的旧标准参数不能进入 active。
5. 每个标准参数必须有字段映射、解析规则、能力标签和证据角色。
6. LLM 低置信输出必须进入人工复核。
7. 发布版本不可变，后续调整生成新版本。

## 8. 与 M03B 的关系

```text
M03A 品类参数语义资产生成
  -> 发布 taxonomy
  -> M03B SKU 参数事实画像生成
```

如果某个品类没有已发布 taxonomy，M03B 必须阻断，不能使用其他品类 taxonomy 或程序默认参数。
