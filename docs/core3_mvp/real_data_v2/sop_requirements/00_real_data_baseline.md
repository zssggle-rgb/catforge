## 5. 明显质量问题

### 5.1 属性缺失

`attribute_data` 共 2843 行，其中 unknown/空值/`-` 约 961 行。缺失必须解释为 unknown，不能当 false。

高覆盖但缺失明显的字段包括：

- `CPU主频`：35 个型号中 34 行缺失。
- `GPU核数`：19 行缺失。
- `HDR`：13 行缺失。
- `IC型号`：32 行缺失。
- `全面屏`：33 行缺失。
- `UI界面`、`HEVC参数`、`主芯片供应商` 等多为缺失。

### 5.2 卖点覆盖不足

结构化卖点只覆盖 5 个型号：

- `55E3NH-PRO`
- `75E3NH-PRO`
- `75E5Q-PRO`
- `85E52S-PRO`
- `85E5S-PRO`

每个覆盖型号有 `卖点1` 到 `卖点13`。

卖点文本含有明显业务结构，如：

- 核心定位。
- 功能价值。
- 情感价值。
- 便捷体验。
- 差异化定位。
- 行业地位。

M04a 应抽取这些结构，但不能把这些标题直接当最终业务结论。

### 5.3 评论维度拆行和重复

`comment_data` 有 62426 行，但只有：

- 34438 个不同 `comment_id`。
- 13514 个不同评论正文 hash。
- 20916 个不同 `comments_segments`。

说明评论表存在维度拆行、重复正文和分段结构。M05 必须去重并保留维度弱标签。

### 5.4 评论维度分布

高频维度包括：

- 空维度：15766 行。
- 送装维保/安装服务/安装整体服务：9835 行。
- 营销服务/产品体验/产品整体评价：5834 行。
- 产品质量/显示画质/画质整体评价：2833 行。
- 产品质量/音频效果/音频整体评价：1514 行。
- 产品设计/结构布局/屏幕尺寸：1381 行。
- 营销服务/产品价格/性价比：961 行。

维度只能作为弱标签，不能直接生成任务、客群、战场或卖点结论。

## 7. 模块设计必须遵守的数据事实

1. 当前只有海信品牌，竞品不能按品牌内外过滤。
2. 85E7Q 是强样例：有量价、参数、评论，但无结构化卖点。
3. 卖点覆盖少，M04a/M04b 必须支持“缺结构化卖点时降级”。
4. 评论行数多但重复和拆行明显，M05/M06 必须以去重评论和句级 evidence 为基础。
5. 属性字段多但 unknown 比例高，M03 必须把 unknown 和 false 严格区分。
6. 市场渠道当前只有线上，但平台有专业电商/平台电商，M07/M13 的渠道重合首版应基于 platform。
7. 所有模块输出都必须可以回溯到上述原始表行或清洗行。
