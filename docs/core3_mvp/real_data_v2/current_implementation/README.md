## 写作规则

后续模块实现文档统一写清楚：

- 模块定位：解决什么问题，不解决什么问题。
- 输入边界：读取哪些表或上游产物，不读取哪些旧表或临时数据。
- 输出边界：写入哪些表，哪些字段是事实，哪些只是质量提示。
- 业务口径：用户如何理解缺失、过滤、冲突、置信度和异常。
- 执行入口：CLI、API、runner 或调度入口。
- 大数据保护：是否分批、是否逐 chunk commit、是否需要避免并发。
- 测试与验证：对应测试文件、205 实跑结论、已知边界。
