## 数据边界

- AC SKU：155 个。
- AC 参数证据：4761 条。

与 TV 草案一致，本草案先对每个原始字段逐项判断，不跳过字段；处理顺序按每 5 个字段一组展开；全部字段判断完成后，再汇总 AC 标准参数与原始字段映射。

## 字段逐项判断

字段状态口径：

- `进入`：可以进入 AC SKU 参数事实画像。
- `派生`：适合作为衍生字段、组合标签或分段标签，不作为优先原始事实字段。
- `排除-无效`：字段有值但值无法支持可用参数事实。
- `特性标记`：对当前 AC 参数源中的能力特性字段，缺失统一按不具备该特性处理，并在画像中保留规则来源。

## 标准参数复核结论

本草案没有保留 `进入-复核` 参数。需要在生成 SKU 参数事实画像时做规则清洗的情况如下：

| 标准参数 | 处理规则 |
|---|---|
| `heating_capacity_w` | 原始值 `0` 按无制热功能处理，制热量值可记为 0，并派生 `heating_function_flag=false` |
| `airflow_volume_m3h` | 原始值 `0` 按异常/未知处理，不作为真实循环风量 |
| `energy_grade_normalized` | 优先使用 `能效等级合并`，用 `能效等级`、`能效` 校验 |
| `product_series` | 合并 `系列` 和 `三大品牌系列`，两者冲突时进入复核池 |

## 排除字段清单

本轮 31 个 AC 原始字段中没有直接排除的字段。`派生` 字段不作为优先核心事实，但保留用于分组、校验或业务分段。

## 后续落地建议

1. 先修复 M00/M02 的品类边界：AC SKU 不应混在 TV 批次下；在修复前，M03 读取 AC 参数时必须按 `sku_code like 'AC%'` 或更可靠的品类标识过滤。
2. AC 标准参数 v0.1 可以先固化为配置；后续新增 AC 数据进来时，只做新字段小范围判断，不全量重建标准参数。
3. M03 生成 AC SKU 参数事实画像时，优先输出核心事实字段；派生/业务分段字段作为辅助标签，不参与核心能力比较。
4. 对 `制热量=0` 按无制热功能处理；对 `循环风量=0` 仍进入低置信/异常值处理，不作为真实循环风量。
