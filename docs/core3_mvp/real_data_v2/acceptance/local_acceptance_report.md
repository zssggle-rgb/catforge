## 4. 验收判断

本地链路可以支撑后续 205 恢复后的真实数据运行。原因是：当前实现已经按原始表、清洗表、证据表、抽取/画像表、结果表和验收/门禁表分层；后续模块消费上一阶段产物；API 和前端只读取 M15/M16 形成的业务结果，不在前端重新推导竞品。

## 5. 剩余风险

1. 本地样例是小规模构造数据，不能替代 205 真实数据水位验收。
2. 205 的 85E7Q 当前真实样例曾存在结构化宣传卖点缺口；恢复后必须把该情况展示为数据缺口和复核提示。
3. 205 后续继续补充 SKU、卖点、评论后，三竞品角色可能变化，必须重跑 M00-M16 并保留新的 run_id。
4. 当前本地验收没有执行真实浏览器截图复核，只用前端单元测试和构建验证页面结构与语言边界。
